# 作家文风 Skill 元工厂

## 核心身份

本 skill 是**母 skill / 元工厂**。

- `style-akutagawa`、`style-dazai`、`style-kafka` 等是本工厂生成的**子产品与测试探针**。
- 子 skill 的目标是接收主题后直接生成高辨识度成稿。
- 母 skill 的目标是稳定制造更好的子 skill，并从所有子产品的表现中学习。
- 优先优化母 skill 的生成方法。只有作者独有问题才写入对应子 skill。

不要把本 skill 退化为文风知识百科、九层分析报告或人工模板集合。分析过程只为提高最终子 skill 的成稿效果服务。

## 用户可见结果

根据请求只交付以下结果之一：

1. 新建或更新后的 `style-<author>/SKILL.md`。
2. 对该子 skill 的真实生成测试结果。
3. 母 skill 的版本改进摘要。

除非用户要求，不展示冗长研究过程、候选规则、内部评分和逐步推理。

## 三种工作模式

### 模式 A：创建作者 Skill

触发示例：

- “给我做一个海明威文风 skill”
- “蒸馏村上春树的写作风格”
- “新增 style-xxx”

输入：作者、可用语料或可访问来源、目标输出语言。

输出：

```text
style-<author>/
├── SKILL.md
├── test-prompts.json
└── results.tsv
```

### 模式 B：优化子 Skill

触发示例：

- “太宰治写得不像”
- “优化 style-kafka”
- “这篇太像网络散文”

输入：子 skill、失败成稿、用户反馈。

输出：通过盲测的新版本；未胜出则保留旧版。

### 模式 C：优化母工厂

触发示例：

- “迭代作家 skill 生成器”
- “把总的作家 skill 做好”
- “让以后生成的作者 skill 都更强”

输入：多个子 skill 的失败模式、测试记录和用户反馈。

输出：本 `SKILL.md` 的可复用改进，以及重新生成或更新子 skill 的迁移计划。

## 总工作流

```text
明确作者与目标
→ 建立母语语料集
→ 提取可验证文风机制
→ 生成子 skill
→ 建立跨题材黄金测试集
→ 运行旧版 / 新版 / 无 skill 基线
→ 独立盲评
→ 保留胜出版本或回滚
→ 聚合多个子 skill 的失败
→ 只把跨作者通用经验回写母 skill
→ 下一轮生成
```

## Phase 1：语料工程

### 1.1 优先级

按以下顺序收集：

1. 作者母语原文。
2. 不同创作时期的代表作品。
3. 不同体裁，但只在目标 skill 需要多体裁时加入。
4. 可靠版本的译文，仅用于理解，不用于推断母语句法。

### 1.2 最低要求

- 作者级 skill：至少 3 部作品或 3 个创作阶段。
- 作品级 skill：可以只用 1 部作品，但 name 和 description 必须标明范围。
- 每条精确统计必须可复算。
- 样本不足时降低置信度，不得把局部特征写成作者铁律。

### 1.3 语料分层

每个结论标记为：

- `corpus_fact`：可从语料复核。
- `stable_pattern`：跨作品重复出现。
- `local_pattern`：只属于特定作品或时期。
- `generation_hypothesis`：等待实测的生成规则。

只有 `stable_pattern` 和经测试胜出的 `generation_hypothesis` 可以进入子 skill 核心执行协议。

## Phase 2：提取文风机制

不要平均分析所有维度。优先找出最能区分该作者的 4-8 个机制：

1. 叙述者是谁，离人物和读者多远。
2. 作品靠什么推进：事件、心理、对话、规则、观察还是论证。
3. 句子如何呼吸：长度、停顿、转折、重复、从句和落点。
4. 情绪如何产生：直说、动作、场景、关系、反讽或省略。
5. 人物如何暴露自己：语言、身体、选择、谎言或他人目光。
6. 开头如何进入，结尾如何停止。
7. 哪些特征是廉价替代品，最容易造成“像但不真”。
8. 换到陌生题材后仍能保留什么。

### 机制筛选规则

一条规则进入子 skill 前必须回答：

- 它是否能直接改变成稿？
- 它是否区别于一般“好文笔”？
- 它是否能在至少两个不同主题复现？
- 删除它后，盲评相似度是否下降？

任一答案为否，不放入核心协议；可放入参考资料或删除。

## Phase 3：生成子 Skill

### 3.1 Frontmatter

```yaml
---
name: style-<author>
description: "<作者>文风生成 skill。说明可生成的体裁、最关键的文风机制，以及用户用作者名要求写作时触发。"
---
```

description 必须同时包含：

- 做什么。
- 何时触发。
- 作者常见中文名、原文名或英文名。
- 生成任务，而不只是“分析文风”。

### 3.2 子 Skill 固定结构

每个子 skill 按此顺序编写：

1. `硬输出契约`
2. `作者母语与翻译协议`
3. `四项独有辨识度指标`
4. `核心生成机制`
5. `体裁路由`
6. `三稿竞争与低分重写`
7. `专属反模式`
8. `最终交付检查`
9. `语料边界与证据`

把长篇统计、原文证据和研究笔记放入 `references/`。`SKILL.md` 只保留运行时真正需要的规则，目标不超过 500 行。

### 3.3 硬输出契约

默认：

- 先用作者主要创作语言独立完成原稿。
- 再提供逐段对应的中文文学翻译。
- 中文作者只输出中文。
- 用户明确要求单一语言时服从用户。
- 第一行直接进入规定格式。
- 不输出寒暄、摘要、技法说明、评分、自检或候选稿。

双语格式：

```text
## [母语]原文
[标题]
[完整正文]

## 中文翻译
[中文标题]
[完整译文]
```

### 3.4 单次成稿算法

```text
读取主题与约束
→ 选择当前任务最相关的 5-8 条规则
→ 内部生成 3 个候选
→ 隐去标签进行比较
→ 选择最高分候选
→ 扫描专属反模式
→ 低于 90 分则针对最低维度重写
→ 最多 3 轮
→ 母语定稿
→ 中文翻译
→ 格式和语言纯度检查
→ 只交付最终稿
```

三个候选必须有不同重心：

- A：叙述声音与句法。
- B：结构与场景推进。
- C：人物心理、关系和结尾。

禁止把三稿逐句拼贴成不统一的成稿。

## Phase 4：作者专用评分器

每个子 skill 先定义 4 项作者独有指标，再使用统一评分：

| 维度 | 权重 |
|---|---:|
| 作者独有辨识度 | 30 |
| 叙述声音与距离 | 15 |
| 结构与推进 | 15 |
| 句法、节奏与词汇 | 15 |
| 人物与情绪机制 | 10 |
| 母语自然度 | 10 |
| 主题完成与原创性 | 5 |

### 交付门槛

- 总分至少 90。
- 用户主题、体裁、长度和语言要求全部满足。
- 母语自然度无硬伤。
- 没有原作复刻。
- 没有命中专属反模式。
- 完整翻译，无摘要替代。

任一硬门槛失败，必须重写，不得用总分抵消。

## Phase 5：专属反模式

每个子 skill 必须包含 5-10 条“廉价相似”黑名单。

示例：

- 太宰治：孤独、雨夜、便利店、无缘无故想哭，不等于告白者的自我表演与羞耻。
- 卡夫卡：官僚、怪事、房间、无解结尾，不等于规则化困境。
- 鲁迅：尖刻、短句、文言虚词、国民性标签，不等于由具体材料产生的反讽。
- 陀思妥耶夫斯基：犯罪、宗教、喊叫，不等于复调冲突和心理压力。
- 叔本华：悲观、格言、长句，不等于命题与论证。

发现廉价替代时，删除表面标志，从叙述机制、结构、句法和人物关系重写。

## Phase 6：黄金测试集

新建 `test-prompts.json`，最终扩展到 15-30 个主题：

- 典型题材：3 个。
- 非典型题材：至少 5 个。
- 现代生活：至少 3 个。
- 关系、行动、场景、观念任务：各至少 1 个。
- 极短、标准、长文：各至少 1 个。
- 只要母语、只要中文、双语：各至少 1 个。
- 诱导复刻原作：至少 2 个。
- “随便主题”：至少 1 个，用于检查模型是否走廉价题材捷径。

测试主题不能大量包含夜晚、雨、死亡、酒、孤独、贫穷等提示性词汇。

## Phase 7：子 Skill 盲测

对每轮候选执行：

1. 固定旧版、候选版、无 skill 基线。
2. 从黄金测试集抽取至少 8 题，非典型题材不少于一半。
3. 三个版本分别生成。
4. 去除版本标签并随机排序。
5. 至少 2 个独立评审评分。
6. 评审只看提示词和成稿，不看 SKILL.md、版本名和预期修改。

候选版必须同时满足：

- 平均分至少 90。
- 比旧版平均高至少 3 分。
- 至少 75% 测试题胜过旧版。
- 母语、原创性、格式零硬失败。
- 非典型题材没有退步。

否则回滚候选版，并将失败原因写入 `results.tsv`。

## Phase 8：母工厂自我迭代

子 skill 是母工厂的实验数据，不是最终知识来源。

### 8.1 反馈归因

每次失败先归类：

| 类型 | 写入位置 |
|---|---|
| 所有或多个作者都出现的格式、翻译、评审问题 | 母 skill |
| 某一语言共同出现的问题，如日语混入中文 | 母 skill 的语言协议 |
| 只属于某作者的廉价模仿或叙述问题 | 对应子 skill |
| 单次随机生成失误，无法复现 | 只记日志，不改规则 |

### 8.2 母 Skill 候选实验

母 skill 每次只修改一个可归因变量，例如：

- 语料筛选方法。
- 机制提取方法。
- 子 skill 模板。
- 黄金测试集构成。
- 盲评标准。
- 输出硬门槛。

然后选择至少 3 位差异明显的作者重新生成或更新子 skill：

- 一位中文或东亚作者。
- 一位欧洲小说家。
- 一位哲学、散文或非小说作者。

只有当母 skill 候选使至少 2/3 子产品提升，且没有任何硬门槛退步时，才保留修改。

### 8.3 母 Skill 评分

| 维度 | 权重 |
|---|---:|
| 子产品平均盲测提升 | 35 |
| 跨作者泛化 | 20 |
| 非典型题材稳定性 | 15 |
| 母语与翻译可靠性 | 10 |
| 子 skill 简洁度和可执行性 | 10 |
| 失败恢复与可追溯性 | 10 |

母 skill 总分低于 90，或只提升一个作者，不视为成功迭代。

### 8.4 经验回写格式

把验证过的跨作者经验写入母 skill：

```text
问题：
影响作者：
失败样本：
候选规则：
盲测结果：
结论：keep / revert
适用范围：
```

不要把具体成稿句子写回母 skill。只保留可迁移的生成机制和检测规则。

## Phase 9：停止条件

### 子 Skill

- 平均盲测至少 92。
- 胜率至少 80%。
- 连续两轮提升低于 1 分。
- 所有硬门槛通过。

### 母 Skill

- 至少 5 位不同语言、不同文体作者的子 skill 平均达到 90。
- 新生成作者的首轮平均分达到 85 以上。
- 连续两轮母 skill 修改未使 2/3 以上子产品提升。

达到停止条件后冻结当前最佳版，等待新语料、模型变化或用户反馈。不要为追求“完美”无限增加规则。

## 失败与恢复

| 触发条件 | 首选处理 | 仍失败时 |
|---|---|---|
| 找不到足够母语语料 | 创建作品级 skill 并标明范围 | 不创建作者级结论 |
| 母语生成不自然 | 简化句法并增加语言纯度检查 | 降级为用户指定语言，不伪装母语 |
| 子 skill 只会典型题材 | 增加非典型黄金测试并删除题材捷径 | 回滚到上一版 |
| 三稿都低于 90 | 针对最低维度重写，最多 3 轮 | 交付当前最佳版时明确未达门槛，或不交付 |
| 候选只提升单个作者 | 写入对应子 skill | 不回写母 skill |
| 评审意见冲突 | 增加第三位独立评审 | 无多数意见则保持旧版 |
| 规则越来越长但效果不升 | 删除未带来盲测增益的规则 | 恢复最近胜出版本 |

## 反例黑名单

不要：

1. 把子 skill 数量当作母 skill 质量。
2. 手工修好一个作者后，就宣称母工厂变强。
3. 用同一上下文自写、自评、自证。
4. 只测试作者典型题材。
5. 把研究笔记全部塞入子 `SKILL.md`。
6. 为提高相似度复用原作句子、人物、情节或独特意象。
7. 先写中文再机械翻译成作者母语。
8. 只看静态规则完整度，不跑真实成稿。
9. 候选未胜出仍保留。
10. 无限迭代而没有停止条件。

## 统一优先级

```text
用户明确要求
> 成稿自然与完整
> 作者独有文风机制
> 母语自然度
> 跨题材稳定
> 表面词汇与题材标志
```

## 日志

母 skill 和子 skill 都使用：

```tsv
timestamp	target	parent_version	child_version	old_score	new_score	win_rate	status	failure_type	note	eval_mode
```

`eval_mode` 只能是：

- `full_blind_test`
- `limited_test`
- `dry_run`

`dry_run` 不能作为保留母 skill 改进的唯一依据。
